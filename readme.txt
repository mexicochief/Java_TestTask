В папке JavaTestLaunch находятся jar-файл, sort-it.exe и папка resources
Из командной строки запускать sort-it.exe с параметрами(параметры соответствуют заданию)
После запуска программы выходной файл сохраняется в папке resources, в ней так же должны находится входные файлы.

Сортировка строк проводится без учета регистра.

Реализация:
Функция uploader(Set<String> path) заполняет ArrayList, который содержит объекты типа BufferedReader для каждого файла, указанных в параметрах программы. Функция ascendingSorting()/descendingSorting() первым делом заполняет объект типа HashMap(где key - номер файла, value - значение строки). И проверяет есть ли ошибка сортировки в первом элементе. Далее значения из этих файлов сравниваются, проверяется соответствует ли найденный элемент порядку сортировки в исходном файле. Если соответствует, то буфер освобождается и элемент записывается в файл. Считывается следующий файл и цикл начинается заново.

Строки сравниваются следующим образом:
Сравниваемые строки преобразуются в массив функцией toInt() в классе MergeSortStr, где каждый элемент представлен в виде его ascii кода.
Функция hasFirstNumLower() сравнивает полученные массив, и выдает true, если массив стоящий первым аргументом меньше.

Если в исходных файлах нарушен порядок сортировки, то элементы которые нарушают прядок не записываются в выходной файл и программа продолжает работу.
Так же отдельной функцией findBreachInOrderD проверятся нарушен ли порядок в первом элементе (В силу особенностей реализации).
